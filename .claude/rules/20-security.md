# 20 — 보안

## 비밀 정보
- 자격증명을 하드코딩하지 않는다. 환경 변수, 시크릿 매니저, 또는 git-ignore된 `.env`에서 읽는다.
- `.env`, `id_rsa`, `~/.ssh/*`, `~/.aws/credentials`, 또는 키처럼 보이는
  파일의 내용을 읽거나 출력하지 않는다.
- 사용자가 채팅에 비밀을 붙여넣더라도 노출된 것으로 취급하지 않는다.
  되돌려 출력하지 않고, 커밋하지 않고, 메모리 파일에 쓰지 않는다.

## 입력 처리
- 신뢰 경계에서 모든 외부 입력을 검증한다.
- 파라미터화 쿼리/프리페어드 스테이트먼트를 사용한다. SQL 문자열 연결 금지.
- 셸 인자를 이스케이프한다. 셸 문자열 조립보다 언어 네이티브 서브프로세스 API를 우선한다.
- 사용자 입력에서 온 경로는 정규화하고 허용 목록과 대조한다. `..` 트래버설은 거부한다.

## 의존성
- 새 의존성은 (a) 명확한 이유, (b) 활발한 유지보수 확인, (c) 사용자 인지가
  없으면 추가하지 않는다.
- 표준 라이브러리로 가능한 일은 표준 라이브러리로.
- 버전은 락파일로 고정한다. 락파일은 손으로 편집하지 않는다.

## 출력
- 비밀, 토큰, 인증 엔드포인트의 전체 요청 본문, PII를 로그에 남기지 않는다.
- 사용자에게 노출되는 에러에서 스택 트레이스와 내부 경로를 제거한다.

## 암호
- 표준 라이브러리 또는 검증된 라이브러리만 사용. 직접 만들지 않는다.
- 최신 알고리즘 사용 (비밀번호: Argon2 / bcrypt, 대칭: AES-GCM 또는
  ChaCha20-Poly1305, 키: Ed25519 / X25519).
