# 00 — 핵심 협상 불가 규칙

다음 규칙은 절대적입니다. 대부분은 hook이 강제하지만, 나머지는 Claude의 책임입니다.

1. **파괴적 셸 명령 금지** — 사용자가 이 대화에서 명시적으로 요청하지 않는 한
   `rm -rf`, `git reset --hard`, `git push --force`, `git clean -fd`,
   `mkfs`, `dd`, `chmod -R 777`, `sudo`, `curl|wget … | sh` 파이프라인은
   절대 실행하지 않는다.
2. **비밀 정보를 코드/로그/커밋/메모리 파일에 두지 않는다.** 토큰, 비밀번호,
   개인키, 고객 PII, 자격증명이 박힌 운영 URL 모두 금지. 환경 변수로 읽고
   이름으로만 참조한다.
3. **락파일 수동 편집 금지** — `package-lock.json`, `yarn.lock`,
   `pnpm-lock.yaml`, `Cargo.lock`, `poetry.lock`, `Pipfile.lock`, `go.sum`은
   패키지 매니저로만 갱신한다.
4. **`.git/`, `node_modules/`, `vendor/`, `dist/`, `build/`, `.venv/`,
   `target/`** 등 생성/의존성 디렉토리에는 쓰지 않는다.
5. **사용자가 이 대화에서 명시적으로 요청하지 않는 한** 커밋, 푸시, PR 생성을 하지 않는다.
6. **hook 우회 금지.** hook이 동작을 막으면 근본 원인을 고친다. 우회 수단을 만들지 않는다.
7. **조용한 실패 금지.** 도구/테스트/빌드가 실패하면 드러내고 진단한다.
   이해하지 못하는 에러를 `2>/dev/null`로 숨기지 않는다.
8. **모르면 멈춘다.** 요구사항/파일/관례가 불명확하면 추측하지 말고
   핵심 질문 하나를 한다.
